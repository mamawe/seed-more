# seed-more Skill

## 一句话定义

不是压缩，是**发散**。把文档中的"种子"和用户已有知识体系建立跨域连接，生成一篇**发芽报告**。

> "得到大脑帮我发的那个芽，我把它种在了自己的土里。长出来的东西，比原来的更好。"

---

## 四步流水线

```
输入文档
  │
  ▼
┌─────────────────────┐
│  Step 1: 提取种子   │  ← 挑出有理论延伸潜力的知识点
└─────────────────────┘
  │
  ▼
┌─────────────────────┐
│  Step 2: 跨域关联   │  ← 优先搜 ima 知识库，补互联网
└─────────────────────┘
  │
  ▼
┌─────────────────────┐
│  Step 3: Aha 提炼   │  ← 找到张力点，用悖论式金句引爆
└─────────────────────┘
  │
  ▼
┌─────────────────────┐
│  Step 4: 金句回响   │  ← 原文金句 + 跨域思想回响
└─────────────────────┘
  │
  ▼
输出：发芽报告（Markdown）
```

---

## Step 1：提取种子

**目标**：从输入文档中识别 5-10 颗有理论延伸潜力的"种子"。

**种子标准**（满足任意一条即入选）：
- 涉及技术革命与社会结构变迁
- 包含可外化的认知模型/思维框架
- 指向人机关系/工具哲学
- 隐含未被言说的悖论或张力
- 与用户知识库已有内容可能形成对话

**输出格式**（每颗种子）：
```
### 种子 N：[种子名称]

- **原文出处**："[直接引用原文，50字内]"
- **种子类型**：[认知模型 / 社会洞察 / 技术哲学 / 工具批判 / 其他]
- **延伸潜力**：[一句话说明为什么这颗种子值得发芽]
```

**执行提示**：
- 不要提取纯事实性内容（如"某年某月发布了某产品"）
- 优先提取那些"说出来觉得对，但细想有问题"的断言
- 种子数量控制在 5-10 颗，宁缺毋滥

---

## Step 2：跨域知识关联

**目标**：为每颗种子找到 2-3 个跨域知识连接点。

**搜索优先级（严格执行）**：
1. **先搜 ima 知识库** → 用 `mcp__ima-mcp__search_knowledge`
2. **知识库无结果** → 用 `WebSearch` 补互联网公开内容
3. **仍无结果** → 基于模型内部知识提供关联，并标注 `[模型知识，未搜索]`

**关联质量的要求**：
- 必须是**跨域**的（心理学种子连哲学，技术种子连社会学）
- 必须指向**具体理论/人物/著作**，不能泛泛而谈
- 必须说明**连接逻辑**（为什么这两个东西有关）

**输出格式**（每颗种子下方追加）：
```
**跨域关联**：

1. [理论/著作/人物] —— [所属学科]
   - 连接逻辑：[一句话说明种子和这个知识的关系]
   - 来源：[ima 知识库 / 互联网 / 模型知识]

2. [理论/著作/人物] —— [所属学科]
   - 连接逻辑：[...]
   - 来源：[...]
```

**ima 搜索操作说明**：
- 需要先调用 `mcp__ima-mcp__get_knowledge_base_list` 获取知识库列表
- 让用户选择一个知识库（或默认用第一个）
- 对每颗种子，用其「种子名称」或「延伸潜力」作为 query 调用 `mcp__ima-mcp__search_knowledge`
- 如果知识库返回了相关内容，优先使用；否则 fallback 到 WebSearch

---

## Step 3：Aha 提炼

**目标**：在种子和关联知识的张力点之间，用一句悖论式金句"引爆"它。

**Aha 金句的特征**：
- **悖论结构**：表面矛盾，细想有道理（"最好的思考工具，是那些让你忘记自己在思考的工具"）
- **14-28 字为宜**，过短没有张力，过长不好记
- **指向行动或认知转变**，不是纯描述
- **避免鸡汤**，要有锋利的洞察感

**输出格式**（每颗种子下方追加）：
```
**Aha 金句**：

> "[悖论式金句]"
>
> —— 张力解析：[1-2 句话解释这个金句为什么成立]
```

**生成方法**：
- 找到种子本身的主张和它的跨域关联之间的**矛盾点**
- 把这个矛盾点用最简洁的方式说出来
- 例：种子说"AI 是工具"，关联知识说"工具会重塑使用者"→ Aha："当你把 AI 当工具使用时，它正在把你变成它的工具。"

---

## Step 4：金句回响

**目标**：从原文找最有穿透力的一句话，再从关联知识中找一句形成"思想回响"。

**输出格式**（每颗种子下方追加）：
```
**金句回响**：

- 📖 原文回响："[从输入文档中找的最有穿透力的一句话]"
- 🌍 跨域回响："[从关联知识中找的对应思想，注明出处]"
```

**要求**：
- 原文回响必须是输入文档中的原话，不能改写
- 跨域回响必须来自 Step 2 找到的关联知识，不能编造
- 两句话之间要有**对话感**（呼应、对立、深化）

## 使用方式

### 方式一：处理 ima 知识库中的文档

1. 调用 `mcp__ima-mcp__get_knowledge_base_list` 获取知识库列表
2. 让用户选择知识库
3. 调用 `mcp__ima-mcp__get_knowledge_list` 获取文档列表
4. 让用户选择要"发芽"的文档
5. 调用 `mcp__ima-mcp__fetch_media_content` 获取文档内容
6. 执行四步流水线

### 方式二：直接粘贴文本

1. 用户直接提供文本内容
2. 执行四步流水线（Step 2 的 ima 搜索仍然执行）

---

## 关键原则

1. **知识库优先**：Step 2 必须先搜 ima 知识库，这是本 Skill 的核心价值
2. **跨域是灵魂**：关联必须在不同学科之间，同域关联没有价值
3. **Aha 要锋利**：金句要有"被击中"的感觉，不能温吞
4. **原文要尊重**：金句回响中的原文回响必须是原话
5. **每个人的发芽报告都不同**：因为每个人的知识库不同，这是个性化来源

## 示例（参考）

以下是我自己跑完全文后部分种子的效果，供参考：

> **种子**："AI 会拉大人与人的差距"
>
> **跨域关联**：祖博夫《监控资本主义》—— 技术革命往往先加剧不平等，再通过制度调整逐步缓解

> - 📖 原文："AI 会拉大人与人的差距。"
> - 🌍 跨域："工具的壁垒不在使用，而在理解。"（克拉克《延伸心灵论》）

## 与得到大脑 original 的差异

| 维度 | 得到大脑「发芽」 | 本 Skill |
|------|----------------|----------|
| 知识来源 | 得到知识库（固定） | 用户自己的 ima 知识库（个性化）|
| 输出格式 | 固定报告 | Markdown，可自定义 |
| 跨域范围 | 预设知识图谱 | 取决于用户知识库 + 互联网 |
| 可复用性 | 一次性 | Skill，可反复使用 |
| 透明度 | 黑盒 | 四步流水线完全透明 |

**核心优势**：每个人的知识库不同，所以每个人的发芽报告都不同。这才是真正的个性化。
